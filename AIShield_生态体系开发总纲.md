### 1.2 核心问题诊断

**致命问题：**
- **零收入零真实用户** — 8天无增长，4个订单全是测试数据
- **检测规则只有竞品的1/7** — 30条 vs AgentAuditKit的221条
- **零分发渠道在运作** — 7个MCP Registry全部未上架
- **数据规模严重不足** — 21个工具 vs 行业8000+已扫描

**结构性问题：**
- 23个服务中仅4个与"Agent安全"相关，其余19个（TTS/ASR/图像/视频/搜索等）与核心定位不符
- services.json中仍使用"atex_official"等旧品牌名
- 代码中大量"ATEX"残留命名
- A2A/生态/Skill市场均为内存存储的演示代码，无持久化
- 竞品已形成三层分工（协议层/通信层/供应链层），AIShield定位模糊

**机会窗口：**
- OWASP MCP Top 10于2026年2月才发布v0.1，对齐即可获权威性
- MCP 2026-07-28规范大版本升级（无状态核心+能力发现+授权加固），创造治理工具需求
- 支付宝3亿笔AI支付、Google AP2等Agent经济基础设施刚刚落地
- Metinc是唯一接近"Agent CA"的第三方（240+评估），市场远未饱和
- 2026年Q1爆发30+个CVE、ClawHavoc事件（1184个恶意skill），安全需求急剧上升

## 第三部分：核心发展原则

### 3.1 三大核心原则

**原则一：自动化（Automation First）**
- 所有可自动化的流程必须自动化：扫描、监控、报告生成、渠道分发
- Agent调用Agent：AIShield自身就是一个Agent，通过MCP/A2A协议被其他Agent发现和调用
- 批量扫描引擎：自动爬取MCP目录 → 批量扫描 → 自动入库 → 自动生成报告
- 持续监控：已扫描工具的版本变更自动检测（Rug Pull检测）
- CI/CD集成：GitHub Action自动扫描PR中的MCP依赖

**原则二：生态化（Ecosystem Thinking）**
- 不做孤立的工具，做生态节点：每个产品既是独立价值，也是生态的入口
- 安全数据飞轮：扫描越多 → 数据越全 → 信任越高 → 用户越多 → 工具越多 → 扫描更多
- 协议标准化：全面支持MCP + A2A + OAuth 2.0，成为协议间的安全桥梁
- 开放合作：与支付方（支付宝/Stripe）、平台方（Coze/Dify）、框架方（LangGraph/CrewAI）集成
- Badge反链病毒循环：被扫描工具嵌入AIShield徽章 → Agent爬取发现 → 调用AIShield → 扫描更多

**原则三：盈利化（Revenue-Driven Iteration）**
- 每个功能上线前必须回答：谁会为这个付费？何时能产生收入？
- 免费层设计原则：覆盖90%个人需求，团队/企业场景必然超限
- "按结果付费"趋势：从按次/按月转向按安全事件拦截数/合规通过率计费
- 平台抽成模式：Phase 3当工具市场成型后，从交易中抽成（参考App Store 15-30%）
- 先有用户再涨价：初始定价低于竞品1/10，积累用户后再逐步提价

### 3.2 四项迭代原则

**原则四：跟踪驱动（Research-Informed）**
- 持续跟踪OWASP MCP Top 10更新、MCP规范演进、A2A协议进展
- 订阅安全事件：CVE数据库、ClawHavoc类事件、MCP SDK漏洞公告
- 竞品监控：Snyk/Cisco/Metinc的产品更新，每月竞品扫描
- 技术雷达：新协议（x402/ACP/UCP/ACT）、新框架（Hermes/BeeAI）的快速评估

**原则五：精益验证（Validate Before Build）**
- 新功能先做最小验证：Landing Page + 手动服务 → 验证需求 → 再自动化
- 不在真空中开发：每个Phase结束做Go/No-Go决策
- 指标驱动：Agent调用量（不是页面PV）是核心指标

**原则六：安全可信（Practice What We Preach）**
- AIShield自身必须通过AIShield扫描
- 所有MCP工具描述不含prompt injection
- Prompt Defense Baseline注入所有对外内容
- 数据最小化：只收集安全审计必需的数据

**原则七：中国市场优先（China-First Global-Ready）**
- 违禁词/合规检测是中国市场独有刚需
- 支付宝/微信支付优先于Stripe
- 中文文档和社区优先
- 架构设计支持国际化，但初期不分散精力

### 4.2 安全层（Phase 1 核心交付物）

**MCP安全扫描引擎 v2.0：**
- 对齐OWASP MCP Top 10，10类 × 5条 = 50+条规则
- 新增Tool Poisoning语义检测（接入LLM做深度分析）
- 新增Rug Pull检测（git diff对比历史版本）
- 新增污点分析（追踪数据流从输入到输出的完整路径）
- 新增实时握手验证（connect → initialize → tools/list 对比声明）
- 批量扫描引擎：自动爬取Smithery/Glama/PulseMCP → 并发扫描 → 入库

**Prompt防火墙 v2.0：**
- 从30条正则升级到100+条规则
- 语义级注入检测（unicode/零宽字符/HTML注释/同形字）
- 工具描述与实际代码行为一致性校验

**中国合规检测（独有差异化）：**
- 违禁词检测（6大平台，持续更新词库）
- 出海合规评估（GDPR/CCPA/数据隐私）
- SEO合规检测
- AI搜索可见度监测

### 4.3 信任层（Phase 2 核心交付物）

**安全评分体系：**
- 对标Metinc 4维评分，扩展为5维：Security / Permissions / Data Handling / Supply Chain / Reliability
- A/B/C/D四级认证，对应Gold/Silver/Bronze/Basic徽章
- OWASP MCP Top 10合规矩阵展示

**Agent身份与信誉：**
- Agent DID（去中心化身份）注册
- 任务完成率/失败率/响应时间/用户评价
- 信誉积分体系（50分初始，成功+5，失败-10）
- 徽章成就系统（熟练/资深/专家Agent）

**持续监控：**
- 已认证工具的版本变更自动检测
- 变更告警推送（邮件/Webhook/Agent通知）
- 安全评分动态更新

### 4.4 经济层（Phase 2-3 核心交付物）

**支付集成：**
- 支付宝（ACT协议2.0对接）
- Creem（国际支付，已集成）
- 按次付费 + 月度订阅 + 企业定制 三档
- Agent自动付费（Agent调用超限 → 自动提示升级 → Agent代用户完成支付）

**工具市场：**
- 安全认证工具的调用网关
- API调用按量计费，AIShield抽成15-30%
- 开发者自助接入，提交工具 → 安全审计 → 上架 → 自动计费

**Agent任务市场：**
- 用户发布任务 → Agent竞价 → 分配 → 执行 → 结算
- 与A2A协议桥接（A2A Task ↔ AIShield Job）
- 评价反馈 → 信誉更新

### 4.5 协议层（贯穿所有Phase）

**MCP协议：**
- 现有MCP Server优化（3工具 → 10+工具）
- 支持MCP 2026-07-28新规范（无状态/能力发现/Tasks/Elicitation）
- 远程模式支持（URL字段，免安装）
- npm包发布（@aishield/mcp-server）

**A2A协议：**
- 兼容Google A2A v1.0规范
- Agent Card注册与发现

**API层：**
- RESTful API（已有）
- OpenAPI 3.0规范（已有）
- Python SDK（pip install aishield）
- OAuth 2.0授权（MCP 2026-07-28要求）

5. **下一个2周做什么？** — 必须服务于"安全+信任+经济"三件事之一

## 第七部分：风险与决策规则

### 7.1 禁止事项

- ❌ 不做通用AI API网关（红海，打不过OpenRouter）
- ❌ 不做MCP目录（打不过Smithery/Glama）
- ❌ 不做与合规安全无关的功能（TTS/ASR/图像/视频/搜索等全部砍掉）
- ❌ 不把AI能力当主打产品（只做安全检测后的增值upsell）
- ❌ 不用"ATEX"或"Exchange"作为品牌名
- ❌ 不在无分发渠道的情况下开发新功能
- ❌ 不在无用户验证的情况下投入超过1周开发时间
- ❌ 不暴露任何API密钥/服务器密码/部署Token

### 7.2 决策优先级（遇到选择时）

| 优先级 | 问题 | 决策 |
|--------|------|------|
| P0 | 新功能是否推进？ | 是否服务于"Agent安全"或"合规工具"？否则不推进 |
| P0 | 先做什么？ | 能产生现金流 + 积累安全数据的优先 |
| P1 | 定价怎么定？ | 参考人工替代成本，定价为其1/10-1/100 |
| P1 | 要砍什么？ | 与"AI合规+Agent安全"无关的功能，砍 |
| P2 | 品牌怎么延伸？ | 一切以"AIShield=安全背书"为核心延伸 |
| P3 | 国际化？ | Phase 1-2聚焦中国，Phase 3再国际化 |
